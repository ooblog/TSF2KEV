⓪-2．インストール/アンインストール＆セットアップ

gitでリポジトリ「https://github.com/ooblog/TSF2KEV.git」を「Clone」するか、
「https://github.com/ooblog/TSF2KEV/archive/master.zip」を「Download ZIP」して展開すると、
「TSF2KEV」フォルダが作成されます。

プログラミング言語「TSF_Tab-Separated-Forth」開発中。
https://github.com/ooblog/TSF2KEV

VimでTSFを編集する場合は「vimsyntax/tsf.vim」を「~/.vim/syntax/tsf.vim」にコピー。
VimでKEVを用いた漢直入力などをする場合は「メニュー→編集(E)→起動時の設定(S)」で「~/.vimrc」を開いて「filetype=tsf」や「:!TSF %」を追記。


   syntax on
   au BufRead,BufNewFile *.tsf set filetype=tsf
   autocmd BufNewFile,BufRead *.tsf nnoremap <F5> :!TSF %<CR>
   command KEV2tsf  :source ~/TSF2KEV/KEV2/kanedit.vim

D言語の環境が用意できる場合「TSF_DMDcompile.sh」のように「TSF」の実行バイナリがコンパイルできます。


   dmd TSF.d TSF_Io.d TSF_Forth.d TSF_Shuffle.d TSF_Calc.d TSF_Time.d TSF_Urlpath.d TSF_Match.d TSF_Trans.d

実行バイナリ「TSF」を環境変数「$PATH」が通るフォルダ(例えばPuppyLinuxの場合「/root/my-applications/bin/TSF」など)にコピーするとLinuxの場合「*.tsf」が実行できるようになるはずです。

①-0．TSF言語の文法、Forthとの差異(ワードをカードと呼ぶ)など。

ForthとかRPN自体初耳な方はwikipediaなどを軽く参考にしてもらって、
「TSF_Tab-Separated-Forth」ではForthとの違いを説明しながら構文や式の概要を説明していきます。

Forth - Wikipedia
https://ja.wikipedia.org/wiki/Forth
逆ポーランド記法 - Wikipedia
https://ja.wikipedia.org/wiki/%E9%80%86%E3%83%9D%E3%83%BC%E3%83%A9%E3%83%B3%E3%83%89%E8%A8%98%E6%B3%95
Gforth マニュアル
http://www.h7.dion.ne.jp/~samwyn/forth/gforth/index.html

あなたが知っている(もしくは今知った)ForthはForth自身でコンパイルされてるかもしれないコンパイラですが、TSFはインタプリタです。実装はD言語だったりPyhtonだったりします。
Forthはワードを定義しますが、TSFはカードの束をスタックと呼び、スタックを呼び出す形になります(スタックの集合をデッキと呼びます)。
Forthの式は逆ポーランド記法ですが、TSFの式は「#TSF_RPN」と「#TSF_calc」系列の２種類あり、括弧や中置記法を用いた式の使用も可能です。
Forthのスタックには型が存在する(演算結果の型チェックが行われていないだけ)と思われるが、TSFは整数も小数も分数も文字列に置き換えます。
Forthはスタックが一本と思われるが、TSFは複数のスタックを使い分けます。

①-1．TSF構文には括弧が存在しません(式では括弧が使えます)。

TSFに存在する構文で用いる記号はタブ文字「\t」による区切りと「#!」によるコメントと、スタック名末尾「:」と関数カード先頭の「#」ぐらいです。

「\t(tab)」
カード(文字列)はタブ文字で区切るので、タブ文字を視認しやすいエディタを用いてください。
「#!(shebang)」
インタプリタ指定「#! /usr/bin/env TSF」を成立させるためにシバン「#!」で始まる行はコメント扱いにして読み飛ばします。
「:(coron)」
スタック名に必須ではありませんが「#TSF_calc」内でスタック名を扱う際に必要。あとLTSVを読み込む「L」副詞で用いる予定。
「#(sharp)」
関数カード名に必須ではありませんが視認性の関係でTSFの関数には頭にシャープが付いてます。

漢直関連で「#」一文字のスタック名を扱うので「#」だけでコメントとか関数カードだとか「:」が無いからスタックじゃないなどと勘違いしないよう注意。

行頭がタブ文字「\t」でもシバン「#!」でもない場合はスタック名の宣言で、行頭がタブ文字の場合はスタックを構成するカード束です。
スタック名とカードの束をまとまて一行で記述(ワンライナー)する事も可能です。

①-2．TSFには変数の型どころか代入したり束縛するような変数自体が存在しません。

TSFは整数も小数も分数もカード(文字列)に置き換えます。カードの束をスタックと呼び、スタックの集合をデッキと呼びます。
#関数カードの演算結果はthatスタックに積み上げられますが、それを変数に代入と呼ぶかは微妙です。
「#TSF_calc」ではスタックを配列に見立てる場合には「#TSF_join[]」を経由せずに式に組み込む事は可能ですす。

①-3．TSFにはサブルーチンの返り値やreturn文が存在しません。return用にタプルを用意するなどの概念もありません。

「#TSF_this」でコールスタックに実行中のスタック名と枚数を記録して次の指定されたスタックが0枚目から実行されます。
thisスタックの数え上げでスタック枚数を越えるオーバーフローが発生するか、「#TSF_countmax」で指定した上限(初期値256)に達すると、スタックを抜けます。
関数型プログラミング言語だとサブルーチンも関数のためthisスタックを抜ける時に値を返すイメージですが、Forthなのでthatスタックに値が積まれる(場合がある)だけです。

①-4．TSFにはfor文やwhile文が無いので再帰がループです。

thisスタック(実行中)のスタック自身を呼ぶ場合はコールスタックに積まずに再度実行されるので、末尾再帰ループとなります。
同様にコールスタックに存在するスタック名を呼び出すとその階層までのスタック名が破棄されます(同名スタックはコールスタックに積まない)のでループとなります。
TSFではコールスタックを直接編集する#関数カードはありません。ループ(再帰)を用いる場合は脱出用の分岐「#exit:」等もセットで用意してください。

①-5．TSFにはif/else文やswitch/case文が無いので分岐は式(比較演算子)などを用います。

「#TSF_this」にドローさせるスタック名を変更するには「#TSF_calc」や「#TSF_casesQON」などを使うのが簡単です。

①-6．スタック代名詞4つのth、this,that,the,theyとハウリングについて。

①-7．カードシャッフル動詞4つのp、peek,poke,pull,pushとドロー＆リターンについて。

※#関数カードの、ドロー(積み下ろし)は「pullFthat」、リターン(積み上げ)は「pushFthat」、してるとも言える。
